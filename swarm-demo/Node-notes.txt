## Docker Swarm "Node" - Simple Explanation

A **node** is a single machine (physical or virtual) that's part of your Swarm cluster.

## Two Types of Nodes

### Manager Node (The Boss)
```bash
# What it does:
- Decides where containers run
- Maintains cluster state
- Handles API requests (docker service create/scale)
- Elects a leader

# How to create:
docker swarm init          # First node becomes manager
```

### Worker Node (The Worker)
```bash
# What it does:
- Only runs containers
- Receives orders from managers
- Cannot make decisions

# How to add:
docker swarm join --token <token> <manager-ip>:2377
```

## Commands to See Nodes:

```bash
# List all nodes
docker node ls

# Output:
# ID                            HOSTNAME        STATUS    AVAILABILITY   MANAGER STATUS
# abc123 *                      macbook-pro     Ready     Active         Leader
# def456                        worker-1        Ready     Active
# ghi789                        worker-2        Ready     Active

# Inspect a node
docker node inspect worker-1

# Promote worker to manager
docker node promote worker-1

# Demote manager to worker
docker node demote worker-1

# Remove a node
docker node rm worker-1
```

---

## In Your YAML - `constraints`:

```yaml
deploy:
  placement:
    constraints:
      - node.role == manager    # Run ONLY on manager nodes
      - node.role == worker     # Run ONLY on worker nodes
      - node.labels.ssd == true # Run on nodes with SSD label
```

## Real Example - Your Setup:

```bash
# You have 1 node (your MacBook)
docker node ls
# ID                            HOSTNAME        STATUS  AVAILABILITY  MANAGER STATUS
# xyz123 *                      docker-desktop  Ready   Active        Leader
```

Your single node is both **Manager AND Worker** вҖ” it can decide AND run containers.
